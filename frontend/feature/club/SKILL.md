# 🏃‍♂️ FEATURE_SKILL.md - Club Domain (Strava-Style)

This document provides the business logic, architectural context, and specific constraints for the "Club" feature. This feature is heavily inspired by **Strava Clubs**, meaning it acts as an activity-driven community rather than just a simple chat group.

---

## 1. Business Logic & Constraints (CRITICAL)

When generating code for Clubs, you MUST enforce the following rules:

### A. Club Creation & Limits
* **Rule:** A user can create a maximum of **5 clubs** where they are the `leader`.
* **Validation:** This must be enforced strictly at the Backend Service layer.

### B. Joining & Privacy
* **Public Clubs:** Users can join instantly. `status` in `CLUB_MEMBERS` becomes `'approved'`.
* **Private Clubs:** Users request to join. `status` becomes `'pending'`. A `club_join_request` notification is sent to the club's Leader/Admins. 

### C. Role-Based Access Control (RBAC)
There are two primary roles in `CLUB_MEMBERS`: `admin` (includes the creator/leader) and `member`.
* **Leader Only:** Can Delete the entire club.
* **Admins Only:** Approve/Reject pending members, Kick/Remove members, Create Events, Post/Pin Announcements.
* **Members:** Can view leaderboards, participate in events, post in the club, and view announcements. Can "Leave" the club.

## 2. Frontend UI/UX Guidelines (Android MVVM)

All screens in this feature are implemented as **Fragments** mounted on the `MainActivity`.

### A. Bottom Navigation "Clubs" Tab (`ClubsFragment`)
* **State 1 (Has Clubs):** If the user is currently in at least one club, display a `RecyclerView` with a **1-item-per-row** layout (Linear) showing "My Clubs".
* **State 2 (No Clubs / Discover):** If the user is not in any club, display a **2-item-per-row** layout (Grid) showing all available/suggested clubs with a prominent "Join" button on each card.

### B. Club Detail Page (`ClubDetailFragment`)
The Detail page uses a `ViewPager2` + `TabLayout` with the following **5 Tabs**:

1. **Newsfeed / Posts (DEFAULT TAB):** * Functions exactly like the Home Newsfeed but filtered ONLY for posts explicitly created within this club (`POST.club_id = this.club_id`).
2. **Overview (Tổng quan):**
   * Displays Club Name, Avatar, Cover, and Bio.
   * Lists the Leader and Admins.
   * **Action Buttons:** "Leave Club" (for Members/Admins). "Delete Club" (ONLY for the Leader).
3. **Activities (Hoạt động):**
   * A feed of workout records (activities) generated by members.
   * **Crucial Logic:** Only show records created *after* the member joined the club (`RECORD.start_time >= CLUB_MEMBERS.joined_at`).
4. **Events (Sự kiện):**
   * Categorized lists of "Upcoming Events" and "Past Events".
5. **Statistics (Số liệu thống kê):**
   * **Aggregated Stats:** Total distance and Total activities of ALL members (accumulated only since they joined the club).
   * **Club Records:** Longest distance, Longest duration across the club.
   * **Personal Bests:** The current user's longest distance and duration within this club.
   * **Leaderboard:** Top 10 members ranked by total distance.

* **Component Reuse:** Always check `:core:system` for `layout_top_app_bar`, primary buttons, and avatar placeholder drawables before creating new ones.

## 3. Database Schema Reference & Updates

The Club feature spans several specific tables. 

**🚨 IMPORTANT SCHEMA UPDATE:** The `POST` table now includes a `club_id` to differentiate between personal posts and club-specific posts.

* **`CLUBS`**: `club_id`, `name`, `description`, `avatar_s3_key`, `privacy_type` ('public'/'private'), `leader_id`.
* **`CLUB_MEMBERS`**: `club_id`, `user_id`, `role` ('admin', 'member'), `status` ('pending', 'approved', 'rejected'), `joined_at`.
* **`CLUB_EVENTS`**: `event_id`, `club_id`, `created_by`, `title`, `description`, `start_time`, `end_time`.
* **`CLUB_EVENT_PARTICIPANTS`**: `event_id`, `user_id`.
* **`POST` (Updated):** `post_id`, `record_id`, `owner_id`, **`club_id` (FOREIGN KEY REFERENCES CLUBS, NULLABLE)**, `title`, `description`, `photo_url`...

*(Note: If `club_id` in `POST` is NULL, it is a personal home-feed post. If it has a value, it belongs to the Club's Newsfeed).*
